## 1. Что такое свёрточная нейронная сеть

**Нейронная сеть** — это математическая модель, вдохновлённая работой мозга. Она состоит из слоёв, каждый из которых выполняет простые операции над числами. Обучение — это автоматический подбор параметров так, чтобы сеть давала правильные ответы.

**Свёрточная нейронная сеть (CNN, ConvNet)** — особый класс сетей для работы с изображениями. Главная идея: вместо того чтобы смотреть на всё изображение сразу, сеть **скользит маленьким окном (фильтром/ядром)** по картинке и ищет локальные признаки — края, текстуры, формы.

### Основные строительные блоки

| Слой | Что делает |
|------|-----------|
| `Conv2d(in, out, 3)` | Свёртка 3×3. Каждый из `out` фильтров «ищет» свой паттерн в изображении. Выход — `out` карт признаков |
| `BatchNorm2d` | Нормализует значения внутри батча. Ускоряет обучение и делает его стабильнее |
| `ReLU` | Функция активации: оставляет положительные числа, обнуляет отрицательные. Вносит нелинейность |
| `MaxPool2d(2)` | Берёт максимум в окне 2×2 и уменьшает карту в 2 раза. Это сжатие — сеть видит более крупные структуры |
| `AdaptiveAvgPool2d` | Усредняет признаки до заданного размера (например 1×1 или 2×2) независимо от входа |
| `Dropout` | Случайно «выключает» нейроны во время обучения. Защита от переобучения |
| `Linear` | Полносвязный слой — финальное решение |

В работе обучены две независимые сети:

```
Кадр с веб-камеры
       │
       ▼
┌──────────────────────────────────────────┐
│  Скользящее окно + пирамида масштабов    │  ← sliding window + image pyramid
│  Вырезаем патчи 64×64                   │

      └─────────────────┘
               │
               ▼

```

## 3. Детектор лиц — FaceDetectorCNN

### Задача

Бинарная классификация: **лицо** или **не лицо** (есть лицо на патче 64×64 — да/нет).

### Архитектура

```python
class FaceDetectorCNN(nn.Module):
    def __init__(self):
        self.net = nn.Sequential(
            Conv2d(3, 16, 3, padding=1), BatchNorm2d(16), ReLU,
            MaxPool2d(2),                                        # 64→32
            Conv2d(16, 32, 3, padding=1), BatchNorm2d(32), ReLU,
            MaxPool2d(2),                                        # 32→16
            Conv2d(32, 64, 3, padding=1), BatchNorm2d(64), ReLU,
            AdaptiveAvgPool2d(1),                               # 16→1
            Flatten(),
            Dropout(0.4),
            Linear(64, 1),                                      # → 1 число
        )
```

### Пошаговое преобразование входа

```
Вход: [B, 3, 64, 64]   — батч из B изображений, 3 канала (RGB), 64×64 пикселя

Блок 1: Conv(3→16) + BN + ReLU   → [B, 16, 64, 64]
         MaxPool(2)               → [B, 16, 32, 32]   ← пространство уменьшилось вдвое

Блок 2: Conv(16→32) + BN + ReLU  → [B, 32, 32, 32]
         MaxPool(2)               → [B, 32, 16, 16]   ← ещё вдвое

Блок 3: Conv(32→64) + BN + ReLU  → [B, 64, 16, 16]
         AdaptiveAvgPool(1×1)     → [B, 64,  1,  1]   ← глобальное среднее
         Flatten                  → [B, 64]

         Dropout(0.4)             ← случайно зануляем 40% нейронов
         Linear(64 → 1)           → [B, 1]            ← одно число — логит
```

**Логит** → `sigmoid` → вероятность от 0 до 1. Если больше порога (0.84) — считаем, что на патче есть лицо.

### Почему такая глубина?

- **Блок 1** (16 фильтров) учит низкоуровневые признаки: края, градиенты цвета.
- **Блок 2** (32 фильтра) комбинирует края → части лица: нос, глаза как абстракции.
- **Блок 3** (64 фильтра) улавливает высокоуровневую структуру — «похоже ли это на лицо в целом».
- `AdaptiveAvgPool(1)` схлопывает пространство в 1×1 — итоговый «вектор признаков» — и это работает для детектора, где нам важно только «есть ли лицо», а не «где именно».

## 4. Классификатор эмоций — EmotionCNN

### Задача

4 класса: `angry`, `happy`, `sad`, `surprised`.

### Входные данные

Изображение конвертируется в **оттенки серого (grayscale)**. Почему:
- Датасет FER2013, на котором обучена сеть, изначально grayscale (лица без цветовой информации).
- Цвет кожи не несёт информации об эмоции — она выражается формой: движение мышц, морщины, форма рта и бровей.
- Канал 1 вместо 3 → модель в 3 раза меньше и быстрее.

### Архитектура

```python
class EmotionCNN(nn.Module):
    def __init__(self, num_classes=4):
        # ── Feature extractor ──────────────────────────────────────
        self.features = nn.Sequential(
            Conv2d(1, 32, 3, padding=1), BN(32), ReLU,          # блок A
            Conv2d(32, 32, 3, padding=1), BN(32), ReLU,         #  "
            MaxPool2d(2), Dropout2d(0.1),                        # 64→32

            Conv2d(32, 64, 3, padding=1), BN(64), ReLU,         # блок B
            Conv2d(64, 64, 3, padding=1), BN(64), ReLU,         #  "
            MaxPool2d(2), Dropout2d(0.1),                        # 32→16

            Conv2d(64, 128, 3, padding=1), BN(128), ReLU,       # блок C
            MaxPool2d(2),                                        # 16→8
        )
        # ── Spatial pooling ────────────────────────────────────────
        self.pool = nn.AdaptiveAvgPool2d(2)                      # 8→2

        # ── Classifier head ────────────────────────────────────────
        self.head = nn.Sequential(
            Flatten(),              # [B, 128, 2, 2] → [B, 512]
            Dropout(0.45),
            Linear(128*2*2, 4),    # → [B, 4]  — 4 эмоции
        )
```

### Пошаговое преобразование входа

```
Вход: [B, 1, 64, 64]   — grayscale, 64×64

═══ Блок A (двойная свёртка) ═══════════════════════════
  Conv(1→32) + BN + ReLU   → [B, 32, 64, 64]
  Conv(32→32) + BN + ReLU  → [B, 32, 64, 64]
  MaxPool(2) + Dropout2d    → [B, 32, 32, 32]

═══ Блок B (двойная свёртка) ═══════════════════════════
  Conv(32→64) + BN + ReLU  → [B, 64, 32, 32]
  Conv(64→64) + BN + ReLU  → [B, 64, 32, 32]
  MaxPool(2) + Dropout2d    → [B, 64, 16, 16]

═══ Блок C (одиночная свёртка) ════════════════════════
  Conv(64→128) + BN + ReLU → [B, 128, 16, 16]
  MaxPool(2)               → [B, 128,  8,  8]

═══ Spatial Pooling ════════════════════════════════════
  AdaptiveAvgPool2d(2×2)   → [B, 128,  2,  2]
  Flatten                  → [B, 512]

═══ Classifier ═════════════════════════════════════════
  Dropout(0.45)
  Linear(512 → 4)          → [B, 4]   — логиты 4 классов
  Softmax (при инференсе)  → [B, 4]   — вероятности (сумма = 1)
```

### Почему AdaptiveAvgPool(2×2), а не (1×1)?

Это ключевое архитектурное решение.

- При `AdaptiveAvgPool(1×1)` — **глобальное среднее** по всей карте 8×8. Теряется пространственная информация: «что вверху лица» и «что внизу» смешивается в одно число.
- Эмоция определяется **пространственно**: брови (верх) движутся по-разному при злости и удивлении, углы рта (низ) — при счастье и грусти. Поэтому важно сохранить хотя бы 2×2 — верх/низ, лево/право.
- `AdaptiveAvgPool(2×2)` → 128×2×2 = 512 чисел — достаточно информации, и сеть остаётся компактной.

### Почему двойные свёртки в блоках A и B?

Это приём из VGG (2014). Две свёртки 3×3 последовательно дают то же **рецептивное поле**, что одна свёртка 5×5, но:
- **Меньше параметров**: 2×(3×3) = 18 весов vs (5×5) = 25 весов
- **Больше нелинейностей**: между ними стоит ReLU, что увеличивает выразительность сети

## 5. Подготовка данных и аугментация

### Данные для детектора лиц

**Позитивные примеры** (лица) — датасет LFW (Labeled Faces in the Wild): ~13 000 фотографий знаменитостей.

**Негативные примеры** (не лица) — фрагменты фоновых изображений из `skimage.data`: кирпичная стена, трава, кофе, гравий и т.д. Это принципиально важно: если брать негативы как «перевёрнутые лица», сеть учится отличать только ориентацию, а не содержание.

### Данные для классификатора эмоций

~19 000 изображений, 4 класса. Источники:
- FER2013 (Facial Expression Recognition 2013) — классический датасет с лицами из интернета
- Реальные фотографии пользователя (`IMG_*.jpg`) — только для **валидации**

### Аугментация (искусственное расширение датасета)

Аугментация — создание новых обучающих примеров из существующих путём случайных преобразований:

```python
# Для детектора (RGB):
RandomHorizontalFlip()              # зеркальное отражение
RandomRotation(15)                  # поворот ±15°
ColorJitter(brightness=0.3, ...)    # изменение яркости/контраста

# Для эмоций (grayscale):
RandomHorizontalFlip()
RandomRotation(20)
RandomAffine(translate=(0.12,), scale=(0.85, 1.15))  # сдвиг + масштаб
ColorJitter(brightness=0.3, contrast=0.3)
RandomAutocontrast(p=0.3)           # авто-контраст с вероятностью 0.3
RandomErasing(p=0.2, scale=(0.02, 0.08))  # случайное «стирание» патча
```

**Нормализация** — вычитаем среднее и делим на стандартное отклонение:
- Детектор: ImageNet-нормализация `mean=[0.485, 0.456, 0.406]`
- Эмоции: `mean=[0.5], std=[0.5]` → значения пикселей в диапазон [-1, 1]

Это важно: нейронные сети сходятся быстрее, когда входные данные центрированы вокруг нуля.

## 6. Обучение: потери, оптимизаторы, расписания

### Функция потерь

**BCEWithLogitsLoss** (для детектора — бинарная классификация):
```
Loss = -[ y·log(σ(ŷ)) + (1-y)·log(1-σ(ŷ)) ]
```
Это бинарная кросс-энтропия. `σ` — сигмоид, переводит логит в вероятность.  
- `y=1` (лицо): хотим `σ(ŷ)` близко к 1
- `y=0` (фон): хотим `σ(ŷ)` близко к 0

**CrossEntropyLoss** (для эмоций — 4 класса):
```
Loss = -log( exp(ŷ_true) / Σ exp(ŷ_i) )
```
Это «мягкий максимум». Сеть должна давать высокий логит правильному классу и низкий — остальным.

### Оптимизаторы

**Adam** (детектор) — адаптивный градиентный спуск. Для каждого параметра автоматически подбирает свой шаг обучения на основе истории градиентов.

**AdamW** (эмоции) — Adam + **weight decay** (регуляризация весов). Небольшой штраф за большие веса: `loss += λ·||w||²`. Это L2-регуляризация, предотвращает переобучение.

### Расписание скорости обучения

**CosineAnnealingLR** (детектор): lr плавно убывает по косинусу от начального значения до ~0.

**OneCycleLR** (эмоции): сначала lr **растёт** (warmup), затем плавно спускается.
```
lr
^
|    /\
|   /  \
|  /    \____
| /           \
+–––––––––––––→ эпохи
  warmup  decay
```
Это помогает «вырваться» из плохих локальных минимумов в начале и аккуратно сойтись в конце. Особенно эффективно на маленьких датасетах.

### Dropout

`Dropout(p=0.4)` — во время обучения случайно зануляет 40% нейронов на каждом шаге. Сеть не может «полагаться» на один нейрон и вынуждена учить избыточные представления. При инференсе все нейроны активны, но их выходы масштабируются на `(1-p)` для сохранения статистики.

## 7. Hard negative mining

После первоначального обучения детектор делает **ложные срабатывания** — находит «лица» на фоне. Техника hard negative mining (классика с времён Viola-Jones, 2001):

1. Прогоняем уже обученный детектор по фоновым изображениям **плотным скользящим окном** (шаг 8 пикселей)
2. Патчи, где сеть **ошибочно предсказала лицо** (score > 0.25) — это «трудные негативы»
3. Добавляем их в обучение и дообучаем ещё 15 эпох

**Почему это работает**: обычные фоновые патчи «лёгкие» — сеть легко их отличает. Трудные негативы — именно те паттерны, на которых сеть путается. Дообучение на них резко снижает ложные срабатывания без потери чувствительности.

## 8. Работа в реальном времени: live_cnn.py

### Параметры системы

```python
CAM_INDEX   = 2      # индекс веб-камеры
THRESHOLD   = 0.84   # порог уверенности детектора
STEP        = 20     # шаг скользящего окна (пиксели)
SCALE_DOWN  = 0.2    # уменьшение кадра для ускорения
WIN         = 64     # размер окна
PYR_SCALE   = 0.80   # коэффициент пирамиды масштабов
IOU_THRESH  = 0.32   # порог IoU для NMS
CROP_PAD    = 0.15   # расширение бокса перед определением эмоции
EMO_SMOOTH_N = 10    # усреднение по последним N кадрам
EMO_MIN_CONF = 0.42  # минимальная уверенность для показа эмоции
```

### Скользящее окно (Sliding Window)

```
┌──────────────────────────┐
│      кадр камеры         │
│  ┌──┐→→→→→→→→→→         │
│  │64│  step=20           │
│  └──┘                    │
│  ↓                       │
│  ┌──┐                    │
│  │  │                    │
│  └──┘                    │
└──────────────────────────┘
```

Каждый вырезанный патч 64×64 подаётся в детектор. Если вероятность > 0.84 → кандидат на лицо.

**Проблема**: лицо может быть любого размера. Решение — **пирамида масштабов**.

### Пирамида масштабов (Image Pyramid)

```
Оригинал (100%)
      │
   ×0.80
      ▼
  80% размера
      │
   ×0.80
      ▼
  64% размера
      │
  ... пока не меньше 64×64 ...
```

На каждом уровне ищем окнами 64×64. Маленькое лицо → находим на крупном масштабе. Большое лицо → находим на маленьком масштабе.

### NMS — Non-Maximum Suppression

Скользящее окно даёт много перекрывающихся боксов для одного лица. NMS оставляет только лучший:

1. Сортируем боксы по уверенности (от высокой к низкой)
2. Берём бокс с максимальной уверенностью — оставляем его
3. Удаляем все боксы, у которых **IoU > 0.32** с лучшим боксом
4. Повторяем с оставшимися

**IoU (Intersection over Union)** — мера перекрытия двух боксов:
```
IoU = площадь пересечения / площадь объединения
```
- IoU = 1.0 → боксы совпадают полностью
- IoU = 0.0 → боксы не пересекаются

### Определение эмоции

После NMS для каждого найденного лица:

1. **Расширяем бокс на 15%** (`CROP_PAD=0.15`) — чтобы захватить всё лицо с контекстом
2. Ресайзим вырезанный фрагмент до 64×64 и конвертируем в grayscale
3. Подаём в EmotionCNN → получаем вектор из 4 вероятностей
4. **Временное сглаживание** — усредняем вероятности по последним 10 кадрам

### Временное сглаживание (Temporal Smoothing)

```python
emo_buf = deque(maxlen=10)  # буфер последних 10 prob-векторов
# ...
emo_buf.append(probs)       # добавляем новый кадр
smooth_probs = np.mean(emo_buf, axis=0)  # среднее
```

**Зачем**: без сглаживания эмоция «дёргается» каждый кадр — angry/happy/angry/happy. Среднее по 10 кадрам (~333 мс при 30 FPS) даёт стабильный результат. Аналог low-pass фильтра.

Дополнительно: если максимальная вероятность < 0.42 (`EMO_MIN_CONF`), показываем «?» вместо названия эмоции — признаём неопределённость.

### Визуализация

Под каждым боксом рисуются горизонтальные полосы — вероятности четырёх эмоций:
```
[ANGRY]   ████████████████         0.72
[HAPPY]   ████                     0.14
[SAD]     ██                       0.09
[SURPR]   █                        0.05
```

## 9. Интересные инженерные решения

### MPS — Metal Performance Shaders (Apple Silicon)

```python
DEVICE = torch.device('mps' if torch.backends.mps.is_available() else 'cpu')
```

На Mac с M1/M2/M3 PyTorch может использовать GPU через Apple Metal вместо NVIDIA CUDA. Обучение ускоряется в 5-10× по сравнению с CPU.

**Ограничение MPS**: `AdaptiveAvgPool2d(k)` требует, чтобы `input_size % k == 0`. Поэтому пул 3×3 с входом 8×8 даёт ошибку (8 % 3 ≠ 0), а пул 2×2 работает (8 % 2 = 0). Это было исправлено в архитектуре эмоций.

### Grayscale vs RGB — domain mismatch

Первоначально EmotionCNN принимала RGB-изображения. Датасет FER2013 grayscale, но хранится как RGB (R=G=B — серые пиксели). При обучении сеть «видела» одинаковые каналы и научилась игнорировать цвет. Но при инференсе с камеры приходит настоящий цвет кожи — **распределение данных изменилось**, и сеть всегда предсказывала «angry».

Решение: явно конвертировать и данные, и инференс в grayscale. Теперь оба распределения совпадают.

### Два отдельных трансформа

```python
infer_det_tf  = T.Compose([ToPILImage, ToTensor, Normalize(ImageNet)])  # для детектора
```
