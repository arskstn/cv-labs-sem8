# Лабораторная работа 5. Классификатор пола (GenderCNN)

**Цель:** обучить свёрточную нейросеть для бинарной классификации изображений лиц по полу (`man` / `woman`) на датасете `new_tpu_dataset`.

**Задачи:**
1. Загрузить и подготовить несбалансированный датасет реальных фотографий лиц
2. Реализовать взвешенную выборку для борьбы с дисбалансом классов
3. Обучить GenderCNN с нуля (без предобученных весов)
4. Оценить качество модели и объяснить результаты

---

## 1. Датасет

**Источник:** `new_tpu_dataset` — реальные фотографии лиц, разбитые по папкам `man/` и `woman/`.

| Класс | Изображений |
|-------|------------|
| man   | 43         |
| woman | 16         |
| **Итого** | **59** |

Датасет **несбалансирован**: соотношение классов ≈ 2.7:1. При наивном обучении сеть могла бы выучить предсказывать всегда `man` и получать ~73% accuracy, ничему не научившись.

**Предобработка:**
- Конвертация в **grayscale** (1 канал): цветовая информация не несёт диагностически значимого сигнала о поле и лишь добавляет вариабельность из-за разного освещения
- Масштабирование до **64×64** пикселей
- Фильтрация: проверка расширения файла, успешности чтения, минимальный размер ≥ 20×20

## 2. Train / Val split и аугментация

Разбиение: **80% train / 20% val** со стратификацией по классам (`train_test_split`, `random_state=42`).  
При 59 изображениях получается примерно **47 обучающих** и **12 валидационных**.

### WeightedRandomSampler

Для борьбы с дисбалансом на этапе обучения применяется `WeightedRandomSampler`: каждому примеру назначается вес, обратно пропорциональный частоте его класса. В результате батч содержит примерно равное число изображений обоих классов независимо от реального соотношения.

### Аугментация (train)

| Преобразование | Параметр | Назначение |
|---|---|---|
| RandomHorizontalFlip | — | инвариантность к ориентации |
| RandomRotation | ±25° | инвариантность к наклону головы |
| RandomAffine | translate 15%, scale 80–120% | инвариантность к масштабу и сдвигу |
| ColorJitter | brightness/contrast ±0.4 | инвариантность к освещению |
| RandomAutocontrast | p=0.4 | нормализация контраста |
| RandomErasing | p=0.25, area 2–10% | имитация частичного закрытия лица |
| Normalize | mean=0.5, std=0.5 | приведение к [-1, 1] |

Аугментация намеренно агрессивная — при всего ~47 обучающих примерах каждая эпоха должна предоставлять модели фактически новые вариации.

## 3. Архитектура GenderCNN

Архитектура идентична EmotionCNN из лаб4 с изменением `num_classes=2`.

```
вход 64×64×1
  Block1: Conv(1→32)+BN+ReLU, Conv(32→32)+BN+ReLU, MaxPool(2), Dropout2d(0.1)
                                                               → 32×32×32
  Block2: Conv(32→64)+BN+ReLU, Conv(64→64)+BN+ReLU, MaxPool(2), Dropout2d(0.1)
                                                               → 16×16×64
  Block3: Conv(64→128)+BN+ReLU, MaxPool(2)                    →  8×8×128
  AdaptiveAvgPool2d(2)                                         →  2×2×128
  Flatten                                                      → 512
  Dropout(0.45)
  Linear(512 → 2)                                              → логиты [man, woman]
```

**Параметры:** ~141 500 обучаемых весов.

**Ключевые архитектурные решения:**
- **Двойные свёрточные блоки** — большее рецептивное поле при том же числе MaxPool-слоёв
- **AdaptiveAvgPool2d(2)** вместо Flatten напрямую — сохраняет грубую пространственную структуру (верхняя/нижняя, левая/правая половина лица) при малом числе параметров в голове
- **Dropout2d(0.1)** между свёрточными блоками — регуляризация карт признаков
- **Dropout(0.45)** перед финальным слоем — основная защита от переобучения

## 4. Обучение

| Гиперпараметр | Значение |
|---|---|
| Оптимизатор | AdamW |
| Learning rate | 3×10⁻⁴ |
| Weight decay | 1×10⁻⁴ |
| Планировщик | OneCycleLR (pct_start=0.1) |
| Эпох | 80 |
| Batch size | 16 |
| Функция потерь | CrossEntropyLoss |

**AdamW** использует L2-регуляризацию корректно (без включения в адаптивные моменты), что важно при ограниченных данных.

**OneCycleLR** — разогрев в первые 10% эпох (lr растёт до max), затем косинусный спад. Это позволяет быстро покинуть неудачные начальные конфигурации и точно сойтись в конце.

Веса наилучшей эпохи (по `val_acc`) восстанавливаются в конце обучения через `model.load_state_dict(best_state)`.

## 6. Почему val_acc = 1.0?

100% точность на валидации — закономерный результат данных условий, а **не признак ошибки или тривиального переобучения**. Три взаимоусиливающие причины:

### 6.1 Критически малый размер val set

При 59 изображениях и разбиении 80/20 валидационная выборка содержит **≈12 примеров**.  
При таком объёме:
- Ошибиться на **1 из 12** = accuracy **91.7%**
- Ошибиться на **0 из 12** = accuracy **100%**

Граница между «идеальным» и «почти идеальным» находится в пределах одного примера. Статистическая погрешность при 12 наблюдениях составляет **±13%** (95% ДИ методом Уилсона) — «100%» и «87%» статистически неотличимы. Никакой вывод о реальном качестве модели сделать нельзя.

### 6.2 Задача бинарная и визуально разрешимая

Классификация пола по лицу проще, чем различение 4 эмоций (лаб4, best val_acc ≈ 72%). Мужские и женские лица имеют систематически различающиеся признаки: форма черепа, ширина скул, надбровные дуги, — которые хорошо представимы даже в grayscale 64×64.

### 6.3 Чистый датасет без фонового шума

`new_tpu_dataset` содержит ровно обрезанные лица без посторонних объектов, без широкой вариации ракурсов или освещения. Это дополнительно упрощает задачу по сравнению с данными «из жизни».

### Вывод о надёжности метрики

Для получения статистически значимой оценки качества потребовалось бы не менее **150–200 валидационных примеров** (95% ДИ width < 7%). При текущем датасете из 59 изображений честная оценка generalization невозможна — val_acc служит лишь сигналом для early stopping, но не реальной метрикой обобщающей способности.

## Вывод

В ходе лабораторной работы реализован и обучен бинарный классификатор пола **GenderCNN**. Повторно использована архитектура EmotionCNN из лаб4 с изменением числа выходных классов — это подтверждает универсальность выбранной архитектуры для разных задач классификации лиц.

Основная техническая сложность лабораторной работы — **несбалансированный и очень маленький датасет**. Она решена комбинацией `WeightedRandomSampler` (выравнивание при обучении) и агрессивной аугментации (искусственное расширение разнообразия). Достигнутый val_acc = 1.0 является ожидаемым артефактом крошечной валидационной выборки (~12 примеров) в сочетании с визуально простой бинарной задачей, а не свидетельством переобучения.
